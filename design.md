##go-linq的并发模型
1. linq的处理被分为不同的步骤，步骤类型包括：

   1.1 select （映射）
   1.2. where （过滤）
   1.3 from （设置数据源）
   1.4 join （连接）
   1.5 Group or distinct（分组）
   1.6 聚合 （折叠)
   1.7 Set (集合的交集、差集、并集、串联）
   1.8 排序
   1.9 获取子分区

2. 每个步骤内部采取数据并行的方式进行处理

4. 每个步骤内的数据并行不采用每个数据一个GO router的简单模式（如果数据太多是不可行的），有2种不同的模式:

   4.1 如果数据是长度已知的列表，则采用数据分割，即分割成数量为（计算机内核数）的块，每块一个GO router
   4.2 如果数据长度未知（比如CHAN），则采用数据分块，即将接受的数据汇集为指定快的大小后，发给GO router处理，每个步骤的go router的数量=（计算机内核数）

3. 步骤之间尽可能不采用串联模式，而是尽量使用管道模式

   3.1 select可以采取管道模式或分区模式（根据数据源的模式）
   3.2 where：结果的数量无法预知，因此要将结果数据分块后以管道模式发送给下个步骤
   3.3 from：分块可以管道，分区无需管道
   3.4 join： 结果的数量无法预知，因此要将结果数据分块后以管道模式发送给下个步骤
   3.5 group or distinct: 结果必须在整个步骤完成后才能得到，因此可以采取分区模式发送给下个步骤
   3.6 聚合： 返回单个值，没有后续处理
   3.7 Set： 除了串联其他都要在整个步骤完成后以分区模式发送，串联可以分块发送
   3.8 获取子分区： 数量已知的分区，其他情况分块
   3.9 排序，与group类似，结果也必须在整个步骤完成后得到
